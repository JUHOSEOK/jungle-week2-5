# Topological Sort (위상 정렬)

## 1. 한 줄 요약
- 위상정렬은 **"먼저 해야 하는 일"을 앞에 두고, 가능한 작업 순서를 차례대로 정하는 방법**입니다.

## 2. 왜 위상정렬이 생겨났을까?
위상정렬은 "아무 순서로나 하면 안 되는 일들"을 다루기 위해 중요해졌습니다.

세상에는 순서가 정해진 일이 많습니다.

- 선수과목을 먼저 들어야 다음 과목을 들을 수 있습니다.
- 조립할 때 밑판을 먼저 만들고 그 위에 부품을 올려야 합니다.
- 프로젝트에서도 설계를 먼저 하고, 그다음 구현하고, 그다음 테스트합니다.

이런 문제는 단순히 번호가 작은 것부터 정렬한다고 해결되지 않습니다.  
진짜 중요한 것은 "**무조건 먼저 끝나야 하는 일**"입니다.

그래서 위상정렬은 이런 필요에서 나왔습니다.

- 순서가 있는 일을 올바르게 처리하고 싶다
- 선행 조건을 어기지 않는 순서를 찾고 싶다
- 전체 일을 가능한 순서대로 나열하고 싶다

쉽게 말하면 위상정렬은 "**순서가 꼬이면 안 되는 일들**"을 정리하려고 생겨났습니다.

## 3. 핵심 개념
### 방향 그래프란 무엇일까?
위상정렬은 **방향 그래프**에서 사용합니다.  
방향 그래프: 선에 방향이 있는 그래프

예를 들어 `A -> B`는  
"A를 먼저 하고, 그다음 B를 할 수 있다"는 뜻입니다.

즉, 양쪽이 똑같은 관계가 아니라 **앞뒤 순서가 있는 연결**입니다.

### DAG란 무엇일까?
위상정렬에서는 **DAG**가 아주 중요합니다.  
DAG: 사이클이 없는 방향 그래프  
쉬운 말로 하면, "화살표를 따라가다 다시 제자리로 돌아오지 않는 그림"

예를 들어:

- `수학 기초 -> 자료구조`
- `자료구조 -> 알고리즘`

이런 것은 괜찮습니다. 순서가 앞으로만 갑니다.

하지만 이런 것은 문제입니다.

- `A -> B`
- `B -> C`
- `C -> A`

이렇게 되면 다시 처음으로 돌아옵니다. 이것을 **사이클**이라고 합니다.  
사이클: 빙글빙글 돌아서 다시 제자리로 오는 연결

### 사이클이 있으면 왜 안 될까?
사이클이 있으면 순서를 정할 수 없습니다.

왜냐하면:

- A를 하려면 C가 먼저여야 하고
- C를 하려면 B가 먼저여야 하고
- B를 하려면 A가 먼저여야 하는 이상한 상황이 되기 때문입니다

즉, 아무것도 먼저 시작할 수 없습니다.

그래서 위상정렬은 **사이클이 없는 방향 그래프**, 즉 DAG에서만 제대로 가능합니다.

### "앞에 해야 할 일을 먼저 끝내는 순서 찾기"라는 뜻
위상정렬은 복잡한 말 같지만, 사실 뜻은 단순합니다.

**앞에 해야 할 일을 먼저 끝내는 순서를 찾는 것**

예를 들어:

- 기초 과목 -> 중급 과목 -> 고급 과목
- 반죽 만들기 -> 굽기 -> 꾸미기
- 설계 -> 구현 -> 테스트

이런 식으로 앞일이 끝난 뒤에만 다음 일을 할 수 있을 때, 그 전체 순서를 정하는 것이 위상정렬입니다.

## 4. 어떻게 동작할까? (진입차수 기반)
위상정렬의 대표 방법 중 하나가 **Kahn 알고리즘**입니다.  
이 방법은 **진입차수**를 사용합니다.

### 진입차수의 뜻
진입차수: 어떤 노드로 들어오는 화살표 개수

쉽게 말하면:

- "이 일을 하기 전에 먼저 끝내야 하는 일이 몇 개 남아 있는가?"를 세는 숫자입니다.

예를 들어:

- `A -> C`
- `B -> C`

라면 `C`는 `A`와 `B`가 먼저 필요하므로 진입차수가 2입니다.

### 진입차수 0인 것부터 시작하는 이유
진입차수가 0이라는 뜻은:

- 이 일을 하기 전에 필요한 일이 없다
- 바로 시작해도 된다

그래서 Kahn 알고리즘은 **진입차수 0인 것부터 큐에 넣고 시작**합니다.

큐: 먼저 들어간 것이 먼저 나오는 줄 서기 통

### 전체 흐름
Kahn 알고리즘은 보통 이렇게 동작합니다.

### 1) 각 노드의 진입차수를 센다
먼저 모든 화살표를 보며  
"이 노드는 먼저 해야 할 일이 몇 개 있지?"를 셉니다.

### 2) 진입차수 0인 노드를 큐에 넣는다
선행 조건이 없는 일부터 시작할 수 있으므로, 그런 노드들을 큐에 넣습니다.

### 3) 큐에서 하나 꺼내서 결과에 넣는다
이제 "할 수 있는 일" 하나를 꺼내서 정답 순서에 추가합니다.

### 4) 그 노드와 연결된 다음 노드들의 진입차수를 줄인다
현재 일을 끝냈으니, 그 다음 일들은  
"먼저 해야 할 일 하나가 해결됐다"라고 볼 수 있습니다.

그래서 연결된 다음 노드들의 진입차수를 1씩 줄입니다.

### 5) 새롭게 진입차수 0이 된 노드를 큐에 넣는다
이제 앞에 해야 할 일이 모두 끝난 노드가 생기면, 그 노드도 시작할 수 있으므로 큐에 넣습니다.

### 6) 큐가 빌 때까지 반복한다
이 과정을 끝까지 반복하면 위상정렬 순서를 얻을 수 있습니다.

아주 짧게 코드로 보면 이런 느낌입니다.

```python
from collections import deque

queue = deque(zero_indegree_nodes)
while queue:
    now = queue.popleft()
    for nxt in graph[now]:
        indegree[nxt] -= 1
        if indegree[nxt] == 0:
            queue.append(nxt)
```

핵심은 이것입니다.

- 지금 바로 할 수 있는 일을 꺼낸다
- 그 일을 끝냈다고 표시한다
- 그래서 이제 할 수 있게 된 다음 일을 큐에 넣는다

## 5. 그림처럼 이해하는 예시
### 예시: 선수과목
다음 과목 관계를 생각해 봅시다.

```text
기초(0) -> 중급(1) -> 고급(3)
기초(0) -> 응용(2)
```

뜻은 이렇습니다.

- `0`을 먼저 들어야 `1`을 들을 수 있습니다
- `0`을 먼저 들어야 `2`를 들을 수 있습니다
- `1`을 먼저 들어야 `3`을 들을 수 있습니다

### 처음 진입차수 세기
- `0`: 0개
- `1`: 1개 (`0`이 먼저 필요)
- `2`: 1개 (`0`이 먼저 필요)
- `3`: 1개 (`1`이 먼저 필요)

진입차수 0인 것은 `0`뿐입니다.  
그래서 먼저 `0`을 큐에 넣습니다.

### 1단계
`0`을 꺼내서 결과에 넣습니다.

결과:

- `[0]`

이제 `0`이 끝났으므로, `0` 다음에 있던 노드들의 진입차수를 줄입니다.

- `1`: 1 -> 0
- `2`: 1 -> 0

이제 `1`, `2`는 시작할 수 있으므로 큐에 넣습니다.

### 2단계
큐에서 `1`을 꺼냅니다.

결과:

- `[0, 1]`

`1`이 끝났으니 `3`의 진입차수를 줄입니다.

- `3`: 1 -> 0

이제 `3`도 큐에 넣을 수 있습니다.

### 3단계
큐에서 `2`를 꺼냅니다.

결과:

- `[0, 1, 2]`

`2` 다음에는 특별히 줄일 것이 없습니다.

### 4단계
큐에서 `3`을 꺼냅니다.

결과:

- `[0, 1, 2, 3]`

이렇게 위상정렬이 끝납니다.

중요한 점은 답이 꼭 하나만 있는 것은 아니라는 점입니다.

예를 들어 `0` 뒤에는 `1`과 `2` 둘 다 가능하므로:

- `[0, 1, 2, 3]`
- `[0, 2, 1, 3]`

둘 다 올바른 답이 될 수 있습니다.

### 다른 비유: 요리 순서
케이크를 만든다고 생각해 봅시다.

- 재료 준비
- 반죽 만들기
- 굽기
- 장식하기

여기서:

- 반죽 만들기 전에는 재료 준비가 먼저
- 굽기 전에는 반죽 만들기가 먼저
- 장식하기 전에는 굽기가 먼저

이런 식으로 "먼저 해야 하는 일"을 지키며 순서를 정하는 것이 바로 위상정렬입니다.

## 6. 위상정렬이 잘 쓰이는 문제
### 1) 선수과목 문제
가장 대표적인 예시입니다.

예:

- 자료구조를 들으려면 프로그래밍 기초가 먼저
- 알고리즘을 들으려면 자료구조가 먼저

이럴 때 어떤 순서로 과목을 들어야 하는지 정할 수 있습니다.

### 2) 작업 순서 정하기
프로젝트나 공장 일처럼 순서가 있는 작업에도 쓰입니다.

예:

- 설계 -> 구현 -> 테스트
- 부품 준비 -> 조립 -> 검사

앞의 일이 끝나야 다음 일을 할 수 있을 때 위상정렬이 잘 맞습니다.

### 3) 의존성 해결
의존성: 어떤 것이 다른 것에 기대고 있는 관계  
쉬운 말로 하면, "이걸 하려면 저게 먼저 필요함"

예:

- 프로그램 A를 설치하려면 라이브러리 B가 먼저 필요
- 파일 X를 만들려면 파일 Y가 먼저 필요

이런 관계를 정리할 때 위상정렬을 씁니다.

### 4) 빌드 순서
빌드: 프로그램을 실행 가능한 형태로 만드는 과정

큰 프로그램은 파일끼리 서로 연결되어 있습니다.  
그래서 어떤 파일을 먼저 만들고, 어떤 파일을 나중에 만들어야 하는지 정해야 합니다.

이럴 때도 위상정렬이 쓰입니다.

## 7. 시간복잡도
위상정렬의 시간복잡도는 보통 **O(V + E)**입니다.

- `V`는 정점의 개수
- `E`는 간선의 개수

시간복잡도: 입력 크기가 커질 때 시간이 얼마나 늘어나는지 나타내는 것

왜 `O(V + E)`일까요?

초보자 눈높이에서는 이렇게 보면 됩니다.

1. 각 정점은 큐에 들어가고 나오는 과정을 많아야 한 번씩 합니다.
2. 각 간선은 진입차수를 줄일 때 한 번씩 확인됩니다.
3. 그래서 전체적으로 노드들과 화살표들을 한 번씩 보는 느낌입니다.

즉,

- 모든 정점을 한 번씩 보고
- 모든 간선을 한 번씩 보며 처리하므로
- 전체 시간은 `O(V + E)`가 됩니다

## 8. BFS/DFS와의 관계
### 위상정렬이 왜 일반 BFS/DFS와는 다른 문제일까?
BFS와 DFS는 보통 "**어떻게 탐색할까?**"에 더 가깝습니다.

- 어디부터 어디까지 갈 수 있는지
- 어떤 순서로 방문할지
- 연결된 곳을 어떻게 찾을지

반면 위상정렬은 "**조건을 지키는 순서를 만들기**"에 가깝습니다.

즉, 그냥 방문만 하는 것이 아니라  
"앞일이 끝난 뒤에만 뒷일을 놓는다"는 규칙이 더 중요합니다.

### 그래도 Kahn 알고리즘에서 큐를 쓰는 이유
Kahn 알고리즘은 큐를 쓰지만, 일반 BFS와 완전히 같은 문제는 아닙니다.

큐를 쓰는 이유는  
"지금 당장 할 수 있는 일들"을 차례대로 꺼내기 위해서입니다.

즉, BFS처럼 가까운 거리 순서를 만들려는 것이 아니라,  
**진입차수 0이 된 노드를 순서대로 처리하기 위해 큐를 쓰는 것**입니다.

### DFS로도 위상정렬이 가능한 이유
DFS로도 위상정렬이 가능합니다.

느낌은 이렇습니다.

- 어떤 노드에서 시작해서 끝까지 들어가고
- 더 갈 곳이 없을 때 결과에 넣고
- 마지막에 순서를 뒤집으면 위상정렬이 됩니다

다만 초보자에게는 **진입차수 기반 Kahn 알고리즘**이 더 직관적인 경우가 많습니다.  
왜냐하면 "먼저 할 수 있는 것부터 꺼낸다"는 생각이 훨씬 눈에 잘 들어오기 때문입니다.

## 9. 초보자가 자주 헷갈리는 점
### 1) 방향 그래프여야 하는 이유
위상정렬은 순서를 정하는 문제이므로,  
"누가 먼저인지"가 보여야 합니다.

그런데 무방향 그래프는 화살표가 없습니다.  
무방향 그래프: 연결만 있고 앞뒤 순서 정보는 없는 그래프

그래서 단순히 이어져 있다는 사실만으로는 "무엇이 먼저인가"를 알 수 없습니다.

### 2) 사이클이 있으면 왜 순서를 정할 수 없는지
사이클이 있으면 서로가 서로를 먼저 하라고 미루는 상황이 됩니다.

예:

- A를 하려면 B가 먼저
- B를 하려면 C가 먼저
- C를 하려면 A가 먼저

그러면 시작점이 없습니다.  
즉, 진입차수 0인 노드가 나오지 않을 수 있습니다.

### 3) 답이 여러 개일 수 있다는 점
위상정렬 답은 하나가 아닐 수 있습니다.

예를 들어:

- A는 먼저
- B와 C는 A 뒤에만 오면 됨

이라면:

- `A, B, C`
- `A, C, B`

둘 다 정답입니다.

### 4) 진입차수 의미
진입차수는 "내 앞에 남아 있는 해야 할 일 개수"처럼 생각하면 쉽습니다.

- 0이면 바로 시작 가능
- 2면 두 가지 일이 먼저 끝나야 함

### 5) 그냥 정렬(sort)과 위상정렬이 왜 다를까?
일반 정렬은 숫자 크기나 알파벳 순서처럼 **기준 하나로 줄 세우는 것**입니다.

하지만 위상정렬은 크기 비교가 아니라  
**선행 조건을 지키면서 순서를 만드는 것**입니다.

예를 들어 숫자 1, 2, 3을 정렬하는 것과  
"1을 끝내야 3을 할 수 있다"는 관계를 정리하는 것은 완전히 다른 문제입니다.

## 10. 문제에서 위상정렬을 떠올리는 신호
문제에서 아래 표현이 보이면 위상정렬을 먼저 의심해 보세요.

### "선행 조건"
이 말은 거의 직접적인 신호입니다.  
앞에 해야 할 일이 있다는 뜻이므로 위상정렬 가능성이 큽니다.

### "먼저 해야 하는 일"
"A를 하기 전에 B를 먼저" 같은 문장이 보이면, 순서 관계가 있는 것입니다.

### "순서를 정하라"
아무 순서가 아니라 **조건을 지키는 순서**를 만들라는 뜻이면 위상정렬을 생각해야 합니다.

### "선수 과목"
과목 문제에서 선수과목이 나오면 거의 대표적인 위상정렬 상황입니다.

### "의존 관계"
의존 관계는 "이게 저것에 기대고 있다"는 뜻입니다.  
즉, 어떤 것이 먼저 준비되어야 다음 것이 가능하다는 뜻이므로 위상정렬 신호입니다.

초보자용으로 정리하면 이런 느낌입니다.

- 순서가 있는 일이다
- 먼저 해야 하는 것이 있다
- 앞일이 끝나야 뒷일을 할 수 있다
- 전체 가능한 순서를 구하라고 한다
- 사이클이 없는 방향 관계처럼 보인다

이런 신호가 여러 개 보이면 위상정렬을 먼저 떠올리면 됩니다.

## 11. 아주 짧은 정리
- 위상정렬은 선행 조건을 지키는 작업 순서를 찾는 방법이다.
- 사이클이 없는 방향 그래프(DAG)에서 사용한다.
- Kahn 알고리즘은 진입차수 0인 것부터 큐에 넣어 처리한다.
- 모든 정점과 간선을 한 번씩 보므로 보통 `O(V + E)`이다.
- 선수과목, 작업 순서, 의존성 문제에서 자주 나온다.

### 암기용 문장 3개
- 위상정렬은 먼저 해야 할 일을 앞에 두는 순서 정하기다.
- 진입차수 0은 지금 바로 할 수 있는 일이다.
- 사이클이 있으면 시작할 수 없어서 위상정렬이 안 된다.
